# Padrao de Estrutura

Este documento define um modelo simples para manter os repositorios organizados e faceis de navegar.

## Estrutura recomendada

```text
nome-do-repositorio/
|-- README.md
|-- LICENSE
|-- docs/
|-- aulas/
|-- exercicios/
|-- projetos/
|-- assets/
`-- .gitignore
```

## Convencoes de nome

- Use nomes em minusculas.
- Separe palavras com hifen.
- Evite nomes longos demais.
- Prefira nomes que expliquem o conteudo da pasta.

## Quando usar cada pasta

| Pasta | Uso |
| --- | --- |
| `docs/` | Anotacoes, guias e documentacao complementar |
| `aulas/` | Conteudo acompanhado durante cursos |
| `exercicios/` | Praticas pequenas e focadas |
| `projetos/` | Aplicacoes ou entregas mais completas |
| `assets/` | Imagens, estilos, fontes e arquivos de apoio |

## Boas praticas

- Mantenha a raiz do repositorio limpa.
- Atualize o README sempre que uma trilha ou projeto importante for adicionado.
- Separe rascunhos e experimentos do conteudo principal.
- Prefira documentacao curta, objetiva e bem linkada.
